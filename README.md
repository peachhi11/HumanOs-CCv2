# HumanOs-CCv2

This branch holds the canonical HumanOS evaluation guidance.

It is for framework-level scorecards and reviewer boundaries: how output quality is judged, what reviewers are allowed to do, and when a correction stays surgical versus being rejected back to composition.

Primary documents in this branch:

- `docs/evaluation/humanos-evaluation-scorecards.md`
- `docs/evaluation/humanos-reviewer-contract.md`

Related docs live on sibling branches:

- `evaluation/reviewer-modes` for reviewer operating modes and scope rules
- `docs/architecture` for the architectural overview
- `architecture/contracts` for runtime and narrative contracts
- `docs/relationship-memory` for relationship-save boundaries
